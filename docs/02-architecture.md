## 数据流

1. `main.py` / `scheduler.py` 触发完整流程。
2. `crawler.py` 读取 `feeds.yaml`，逐个源抓取，输出标准化 item。
3. `extractor.py` 清洗摘要并尝试抓取正文。
4. `classifier.py` 根据文本内容判定分类。
5. `summarizer.py` 生成结构化摘要。
6. `storage.py` 将 item 写入 SQLite，并将未向量化的记录异步向量化到 Chroma。
7. `report.py` 读取 SQLite 生成 Markdown 报告。
8. `app.py` 读取 SQLite 和 Chroma 提供交互式看板。

## 模块职责

| 模块 | 职责 |
|------|------|
| `feeds.yaml` | 定义源名称、URL、类型、启用状态、最大抓取条数 |
| `crawler.py` | 抓取并解析 RSS/Atom，生成标准化 item |
| `extractor.py` | HTML 清洗、正文抽取、内容长度控制 |
| `classifier.py` | 关键词规则分类、置信度计算、标签提取 |
| `summarizer.py` | LLM 结构化摘要、本地降级摘要 |
| `storage.py` | SQLite 持久化、Chroma 向量化、语义检索 |
| `report.py` | 按分类聚合生成日报/周报 Markdown |
| `app.py` | Streamlit 可视化看板 |
| `main.py` | 单次完整流程入口 |
| `scheduler.py` | 定时调度入口 |

## 可扩展点

- 新增源：修改 `feeds.yaml` 即可。
- 新增分类：修改 `classifier.py` 的 `CATEGORY_RULES` 和 `SOURCE_BIAS`。
- 更换 LLM：修改 `.env` 中的 `OPENAI_BASE_URL` 和 `OPENAI_MODEL`。
- 更换 Embedding 模型：修改 `storage.py` 中的 `EMBEDDING_MODEL`。
- 新增输出格式：扩展 `report.py` 或新增导出模块。
