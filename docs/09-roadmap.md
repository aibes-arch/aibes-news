# 路线图

## 第一阶段：最小可用（已完成）

- [x] RSS/Atom 源配置与采集
- [x] HTML 清洗与正文抽取
- [x] 关键词分类与标签
- [x] LLM 摘要 + 本地降级
- [x] SQLite + Chroma 存储
- [x] 日报/周报 Markdown 生成
- [x] Streamlit Web 看板
- [x] 定时调度入口

## 第二阶段：稳定性与效率

- [ ] 增加请求重试与指数退避
- [ ] 增加代理支持（HTTP_PROXY / SOCKS）
- [ ] 源健康度监控与自动禁用
- [ ] 增量采集：只抓取上次运行后新发布的内容
- [ ] 断点续传与异常恢复
- [ ] 更精确的中文分词与关键词匹配

## 第三阶段：智能化提升

- [ ] 使用 LLM 对低置信度文章二次分类
- [ ] 实体识别（公司、产品、人物）
- [ ] 情感分析
- [ ] 热点趋势统计与可视化
- [ ] 相似文章去重（基于向量相似度）
- [ ] 自动聚类生成专题

## 第四阶段：输出与集成

- [ ] 邮件推送日报
- [ ] 微信公众号 / 企业微信推送
- [ ] Slack / Discord / Telegram Bot
- [ ] RESTful API 服务
- [ ] 用户订阅与个性化推荐

## 第五阶段：规模化

- [ ] 分布式采集（Celery / RQ）
- [ ] 支持更多非 RSS 源（Twitter/X、Reddit、Newsletter）
- [ ] 多租户支持
- [ ] 更强大的向量库（Qdrant / Milvus / Pinecone）
- [ ] 前端看板升级为 Next.js / React

## 优先级建议

对于个人或小型团队，建议优先完成：

1. 稳定性增强（重试、代理、增量采集）。
2. 微信/邮件推送。
3. 相似文章去重。
4. 热点趋势可视化。
